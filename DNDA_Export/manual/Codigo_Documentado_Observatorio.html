🏛️ Arquitectura Hexagonal
Cada slice (practicas, cargas, chat) contiene tres capas: dominio (reglas del negocio puras), aplicación (casos de uso) e infraestructura (adaptadores HTTP, repositorios SQLite, proveedores externos). Esto garantiza que las reglas de negocio no dependan de ningún framework externo.
🤖 Patrón RAG para la IA
El chat con IA utiliza Retrieval-Augmented Generation: antes de enviar el mensaje del docente al modelo Llama 3.3, el sistema consulta las estadísticas reales del observatorio y las inyecta en el prompt del sistema. Esto elimina alucinaciones y garantiza respuestas fundamentadas en datos reales.

Backend · Node.js
Propósito: Es el punto de entrada del servidor. Aquí se inicializa la base de datos SQLite, se cargan todos los controladores de cada slice funcional (auth, prácticas, cargas, analíticas, chat) y se configura el servidor Express con sus middlewares globales. JavaScript · Node.js
/**
 * app.js — Punto de entrada del backend.
 *
 * Registra los controladores de cada slice y configura Express.
 * La base de datos se inicializa de forma async antes de arrancar.
 */
// Carga las variables de entorno desde el archivo .env del backend
require('dotenv').config({ path: require('path').join(__dirname, '../.env') });

const express = require('express');  // Framework web
const cors    = require('cors');     // Permite peticiones desde el frontend en otro dominio
const path    = require('path');

// Singleton de la base de datos — se inicializa una sola vez
const db = require('./compartido/infraestructura/BaseDeDatos');

/**
 * Función principal asíncrona.
 * Espera a que SQLite cargue el WASM y cree las tablas
 * ANTES de registrar cualquier ruta HTTP.
 */
async function iniciar() {
  /* ── 1. Inicializar la base de datos ── */
  await db.init();

  /* ── 2. Importar controladores DESPUÉS de que la BD esté lista ──
     Cada controlador instancia sus propios repositorios e inyecta la BD */
  const controladorAuth       = require('./auth/infraestructura/ControladorAuth');
  const controladorPracticas  = require('./practicas/infraestructura/ControladorPracticas');
  const controladorCargas     = require('./cargas/infraestructura/ControladorCargas');
  const controladorAnaliticas = require('./analiticas/infraestructura/ControladorAnaliticas');
  const controladorChat       = require('./chat/infraestructura/ControladorChat');
  const manejadorErrores      = require('./compartido/infraestructura/ManejadorErrores');

  const app  = express();
  const PORT = process.env.PORT || 3000;

  /* ── 3. Middlewares globales ── */
  app.use(cors());           // Habilita CORS para Netlify → Railway
  app.use(express.json());   // Parsea el body JSON de las peticiones

  /* ── 4. Registro de rutas por slice funcional ── */
  app.use('/api/auth',      controladorAuth);       // POST /api/auth/login
  app.use('/api/practices', controladorPracticas);  // GET  /api/practices
  app.use('/api/uploads',   controladorCargas);     // POST /api/uploads/upload
  app.use('/api/analytics', controladorAnaliticas); // GET  /api/analytics/dashboard
  app.use('/api/chat',      controladorChat);       // POST /api/chat

  app.use(manejadorErrores); // Captura todos los errores no tratados

  /* ── 5. Arrancar el servidor ── */
  app.listen(PORT, () => {
    console.log(`✔ Observatorio corriendo en http://localhost:${PORT}`);
  });
}

// Lanzar y capturar errores fatales de arranque
iniciar().catch(err => {
  console.error('Error fatal al iniciar:', err);
  process.exit(1);
});
Decisión arquitectónica: Los controladores se importan dentro de la función iniciar() y no al principio del archivo. Esto garantiza que cuando Node.js ejecuta el require() de cada controlador, la base de datos ya está disponible y las tablas ya han sido creadas.

BaseDeDatos.js
backend/src/compartido/infraestructura/BaseDeDatos.js
Infraestructura · SQLite
Propósito: Es el adaptador de base de datos del sistema. Encapsula la librería better-sqlite3 detrás de una interfaz asíncrona compatible con el patrón de consultas anterior (PostgreSQL). Al inicializarse, crea el archivo físico proyecto_paz.sqlite y ejecuta el DDL completo para crear las tres tablas del sistema si no existen.
JavaScript · SQLite
const Database = require('better-sqlite3'); // Driver nativo SQLite para Node.js
const path = require('path');

class BaseDeDatosWrapper {
  constructor() { this._db = null; }

  /** Abre o crea el archivo SQLite y ejecuta el esquema DDL */
  async init() {
    if (this._db) return this; // Patrón Singleton: no inicializar dos veces
    // El archivo se crea automáticamente si no existe
    const dbPath = path.join(__dirname, '../../../../proyecto_paz.sqlite');
    this._db = new Database(dbPath, { verbose: null });
    // WAL = Write-Ahead Logging: mejora el rendimiento en escrituras concurrentes
    this._db.pragma('journal_mode = WAL');
    this._inicializarEsquema();
    return this;
  }

  /**
   * Emula la API de pg: await db.query('SELECT...', [params])
   * Devuelve { rows: [...], rowCount: N, insertId: N }
   * Los repositorios usan '?' como marcador de posición (SQLite)
   */
  async query(text, params = []) {
    const stmt = this._db.prepare(text);
    // Detectar tipo de sentencia para usar .all() o .run()
    const isSelect = /^\s*(SELECT|PRAGMA|WITH.*SELECT)/i.test(text);
    if (isSelect) {
      const rows = stmt.all(...params);
      return { rows, rowCount: rows.length };
    } else {
      const info = stmt.run(...params);
      // insertId es el ROWID del último INSERT, útil para AUTOINCREMENT
      return { rows: [], rowCount: info.changes, insertId: info.lastInsertRowid };
    }
  }

  /** Devuelve un pseudo-cliente para transacciones explícitas BEGIN/COMMIT/ROLLBACK */
  async getClient() {
    return { query: (text, params) => this.query(text, params) };
  }

  /**
   * DDL: Crea las tres tablas principales del sistema.
   * Se ejecuta con IF NOT EXISTS para ser idempotente.
   *
   * Tablas:
   *   uploads   → registro de cada archivo Excel cargado
   *   practices → cada fila de práctica educativa territorial
   *   analisis_ia → cache de los análisis generados por IA
   */
  _inicializarEsquema() {
    this._db.exec(`
      CREATE TABLE IF NOT EXISTS uploads (
        id            INTEGER PRIMARY KEY AUTOINCREMENT,
        original_name TEXT    NOT NULL,
        stored_name   TEXT    NOT NULL,
        records_count INTEGER NOT NULL DEFAULT 0,
        uploaded_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
        publicado     INTEGER DEFAULT 0     -- 1 = visible en la vista pública
      );
      CREATE TABLE IF NOT EXISTS practices (
        id            INTEGER PRIMARY KEY AUTOINCREMENT,
        upload_id     INTEGER NOT NULL,
        institucion   TEXT, departamento TEXT, municipio TEXT,
        nombre_practica TEXT, conflictos_tipo TEXT,
        politicas_relacionadas TEXT, temas_catedra_paz TEXT,
        latitud REAL, longitud REAL,   -- coordenadas para el mapa de calor
        ...                            -- resto de columnas pedagógicas
        FOREIGN KEY(upload_id) REFERENCES uploads(id) ON DELETE CASCADE
      );
    `);
  }
}

// Exportar instancia única (Singleton)
const inst = new BaseDeDatosWrapper();
module.exports = inst;
Nota de mantenimiento: SQLite requiere ? como marcador de parámetros. Si se migra de vuelta a PostgreSQL, los repositorios deberán usar $1, $2... y el método query() de este archivo deberá adaptarse.

RepositorioPracticasSqlite.js
backend/src/practicas/infraestructura/
Patrón Repository
Propósito: Implementa todos los accesos a la tabla practices de la base de datos. Sigue el patrón Repository: el dominio declara una interfaz (RepositorioPracticas) y esta clase la implementa concretamente para SQLite. Incluye búsqueda paginada con filtros múltiples, inserción masiva de registros desde Excel, actualización de coordenadas y eliminación en cascada.
JavaScript · SQLite
class RepositorioPracticasSqlite extends RepositorioPracticas {
  constructor(db) { super(); this.db = db; }

  /**
   * buscarTodas() — Consulta paginada con filtros dinámicos.
   * Construye el WHERE dinámicamente para evitar inyección SQL
   * usando parámetros preparados (prepared statements).
   *
   * @param {objeto} consulta — { busqueda, departamento, tipo, formacion,
   *                              pagina, tamanioPagina, campoOrden, idCarga }
   */
  async buscarTodas(consulta) {
    // Validar campo de orden contra lista blanca (evita SQL injection)
    const columna   = COLUMNAS_ORDEN.includes(consulta.campoOrden)
                      ? consulta.campoOrden : 'institucion';
    const direccion = consulta.direccionOrden === 'DESC' ? 'DESC' : 'ASC';

    const condiciones = ['1=1']; // Base siempre verdadera
    const params = [];

    // Búsqueda de texto libre en múltiples columnas
    if (consulta.busqueda) {
      condiciones.push(`(institucion LIKE ? OR nombre_practica LIKE ? OR
         municipio LIKE ? OR departamento LIKE ? OR responsables LIKE ?)`);
      const term = `%${consulta.busqueda}%`;
      params.push(term, term, term, term, term);
    }
    // Si no se especifica idCarga, usar siempre la carga más reciente
    if (consulta.idCarga) {
      condiciones.push('upload_id = ?'); params.push(consulta.idCarga);
    } else {
      condiciones.push('upload_id = (SELECT MAX(id) FROM uploads)');
    }

    const desplazamiento = (consulta.pagina - 1) * consulta.tamanioPagina;
    // Paginación mediante LIMIT + OFFSET
    const sql = `SELECT * FROM practices WHERE ${where}
                  ORDER BY ${columna} ${direccion}
                  LIMIT ? OFFSET ?`;
    const res = await this.db.query(sql, [...params, consulta.tamanioPagina, desplazamiento]);
    const total = await this.db.query(`SELECT COUNT(*) as n FROM practices WHERE ${where}`, params);
    return { registros: res.rows, total: parseInt(total.rows[0].n, 10) };
  }

  /**
   * insertarMasivo() — Inserta N prácticas en una transacción atómica.
   * Si falla cualquier INSERT, se hace ROLLBACK completo.
   */
  async insertarMasivo(idCarga, practicas) {
    const client = await this.db.getClient();
    try {
      await client.query('BEGIN');
      for (const f of practicas) {
        await client.query(sqlInsert, [idCarga, f.fecha, f.institucion, ... ]);
      }
      await client.query('COMMIT');
    } catch (err) {
      await client.query('ROLLBACK'); // Garantiza atomicidad
      throw err;
    }
  }

  /** Elimina las coordenadas de una práctica del mapa de calor */
  async actualizarCoordenadas(id, lat, lng) {
    await this.db.query('UPDATE practices SET latitud=?, longitud=? WHERE id=?', [lat, lng, id]);
  }
}
Seguridad: El método buscarTodas() construye el SQL dinámicamente pero siempre usa prepared statements (?) para los valores, nunca concatenación directa de strings. Los nombres de columnas para ORDER BY se validan contra una lista blanca antes de incluirlos en la consulta.

JavaScript
// marcarPublicado: desactiva TODAS las cargas y activa solo la seleccionada.
// Usa una transacción para garantizar que no haya inconsistencias.
async marcarPublicado(id) {
  const client = await this.db.getClient();
  try {
    await client.query('BEGIN');
    await client.query('UPDATE uploads SET publicado = 0');    // Desactiva todas
    await client.query('UPDATE uploads SET publicado = 1 WHERE id = ?', [id]); // Activa solo la elegida
    await client.query('COMMIT');
  } catch (e) { await client.query('ROLLBACK'); throw e; }
}

// crear: inserta un registro de carga y devuelve su ID (lastInsertRowid)
async crear({ nombre_original, nombre_guardado, cantidad_registros }) {
  const res = await this.db.query(
    `INSERT INTO uploads (original_name, stored_name, records_count) VALUES (?, ?, ?)`,
    [nombre_original, nombre_guardado, cantidad_registros]
  );
  return res.insertId; // ID generado por AUTOINCREMENT
}

ManejadorEnviarMensaje.js
backend/src/chat/aplicacion/
Caso de Uso · RAG Pattern
Propósito — Patrón RAG (Retrieval-Augmented Generation): Este caso de uso implementa la lógica central del asistente de IA. Antes de enviar el mensaje al modelo Llama 3.3, el sistema consulta y recupera los datos estadísticos reales del observatorio (analíticas del dashboard + muestra de 10 prácticas) y los inyecta en el prompt de sistema. Esto "fundamenta" las respuestas del modelo en datos reales, eliminando las alucinaciones.
JavaScript · RAG Pattern
async ejecutar(comando) {
  // PASO 1 — RETRIEVAL: Recuperar datos reales de la BD antes de llamar a la IA
  const analiticas = await this.proveedorContexto.obtenerAnaliticas();
  const muestras   = await this.proveedorContexto.obtenerMuestraPracticas(10);

  // PASO 2 — AUGMENTATION: Construir el prompt enriquecido con datos reales
  const promptSistema = this._construirPromptSistema(analiticas, muestras);

  // Solo los últimos 6 turnos del historial (evita tokens excesivos)
  const historialReciente = comando.historial.slice(-6);

  // PASO 3 — GENERATION: Enviar a Groq/Llama con el contexto enriquecido
  const mensajes = [
    { role: 'system', content: promptSistema }, // Contexto con datos reales
    ...historialReciente,                          // Memoria de la conversación
    { role: 'user', content: comando.mensaje },  // Pregunta actual del docente
  ];
  return this.proveedorIA.enviarMensaje(mensajes);
}

/**
 * _construirPromptSistema() — Genera el prompt del sistema que inyecta
 * todos los datos estadísticos reales del observatorio al modelo IA.
 *
 * El modelo recibe: total de prácticas, distribución por departamento,
 * tipos de conflicto, políticas relacionadas, temas de Cátedra de Paz
 * y una muestra de 6 prácticas reales con sus obstáculos.
 */
_construirPromptSistema(analiticas, muestras) {
  const { kpis, porDepartamento, porConflictos } = analiticas;
  return `Eres un asistente experto en análisis de prácticas educativas
sobre paz, memoria y derechos humanos en Colombia.
Tienes acceso a los datos REALES del Observatorio:

RESUMEN: ${kpis.total} prácticas · ${kpis.departamentos} departamentos
TOP DEPARTAMENTOS: ${porDepartamento.slice(0,5).map(r=>`${r.label}: ${r.count}`).join(', ')}
TIPOS DE CONFLICTO: ${porConflictos.slice(0,5).map(r=>r.label).join(', ')}

MUESTRA DE PRÁCTICAS REALES:
${muestras.slice(0,6).map(p=>`• "${p.nombre_practica}" — ${p.municipio}, ${p.departamento}`).join('\n')}

Responde SIEMPRE en español. Usa los datos para fundamentar las respuestas.`;
}
¿Por qué RAG y no fine-tuning? El fine-tuning (ajustar el modelo con datos propios) requiere miles de ejemplos, semanas de cómputo y costos elevados. RAG permite que el modelo responda con los datos actuales del observatorio simplemente inyectándolos en cada consulta. Si se cargan 500 nuevas prácticas, la IA las incorpora automáticamente en la siguiente consulta sin ningún re-entrenamiento.

🔄 Estado centralizado en App
El componente App mantiene el estado global: qué tab está activo, el total de prácticas para el header y el ID de carga para el filtro de la tabla. Los componentes hijos reciben todo por props, lo que facilita el flujo de datos unidireccional de React.
⚡ key={refresco} — Patrón de refresco
Cuando el usuario sube un nuevo archivo Excel, el callback refrescarDatos incrementa el contador refresco. Al pasar ese valor como key al Dashboard, React desmonta y re-monta el componente, forzando una nueva llamada a la API para actualizar los indicadores.

// La URL base se toma de la variable de entorno VITE_API_URL.
// En Netlify apunta al backend en Railway. En local usa localhost:3000.
const BASE = import.meta.env.VITE_API_URL || '/api';

// Wrapper centralizado: todos los fetch del frontend pasan por aquí.
// Si el servidor retorna un error HTTP, lanza una excepción con el mensaje del servidor.
async function solicitud(ruta, opciones = {}) {
  const respuesta = await fetch(`${BASE}${ruta}`, opciones);
  if (!respuesta.ok) {
    const cuerpo = await respuesta.json().catch(() => ({}));
    throw new Error(cuerpo.error || `Error ${respuesta.status}`);
  }
  return respuesta.json();
}
export const enviarArchivo = (ruta, archivo) => {
  const formData = new FormData();
  formData.append('file', archivo);
  // Nota: NO poner Content-Type manual; el browser lo añade con el boundary correcto
  return solicitud(ruta, { method: 'POST', body: formData });
};
